# Few-Shot 示例 — Data Engineer Agent

## Example 1: CHARLS 变量可用性评估

### Input

```
任务: 评估 CHARLS 各 wave 中衰弱相关变量的可用性

项目需求:
  - Fried Phenotype 5 项: 体重下降、疲乏、握力、步速、体力活动
  - 候选预测因子: 人口学、慢性病、功能、认知、心理、实验室
  - 预测窗口: 2 年

数据路径: /Users/wuyouhang/Documents/trae_projects/related to Sarcopenia/charls/
请评估各 wave 的变量覆盖情况并给出数据使用建议。
```

### Reasoning (Chain of Thought)

```
1. 先查看各 wave 有哪些文件 → 主要是 Health_Status_and_Functioning + Biomarker + Blood
2. 对于每个 wave, 检查关键变量是否存在
3. 对于 Fried Phenotype 5 项, 逐项检查每个 wave 的可用性
4. 输出覆盖矩阵 + 缺失率预估 + 建议
```

### 推荐数据方案
```
主分析:    2013 (基线) → 2015 (结局)     ← 最佳
时间验证:  2015 (基线) → 2018 (结局)     ← 独立时间窗
敏感性:    2011 (基线) → 2013 (结局)     ← 不含握力版
```

### 变量缺失率预估 (2013 wave, N≈15,000)
| 类别 | 变量数 | 预估完整度 | 原因 |
|------|--------|-----------|------|
| 人口学 | 6 | >95% | 核心问卷, 几乎无缺失 |
| 功能 (握力/步速) | 4 | 70-80% | 需到场测量, 约20%未测或无法完成 |
| 认知 (MMSE) | 1 | 85-90% | 约10%代理受访或拒绝 |
| 心理 (CES-D) | 1 | 90% | 部分受访者不回答心理问题 |
| 慢病 (自报) | 15 | >90% | 少量"不知道" |
| 实验室 (血液) | ~10 | **50-60%** ⚠️ | 仅同意抽血的受访者 |
| 体格 (BMI) | 1 | 75% | 需测量身高体重 |

### 数据质量注意事项
1. **血液数据缺失非随机**: 更健康的人更愿意抽血 → 选择偏倚
2. **握力/步速缺失**: 可能与衰弱本身相关 (太衰弱的老人才无法完成) → 非随机缺失
3. **代理受访**: ~5-10% 为代理受访, 功能和认知数据需特别标注
4. **各 wave 变量名可能不同**: 2011 使用小写(da049), 2013+ 使用 PascalCase
   → 需要 inspect_headers.py 逐 wave 确认
