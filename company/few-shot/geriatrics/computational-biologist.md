# Few-Shot 示例 — Computational Biologist Agent

## Example 1: 设计衰弱预测建模方案

### Input (from Orchestrator)

```
项目: 基于 CHARLS 预测社区老年人 2 年衰弱转换

临床定义 (from clinical-researcher):
  - 结局: Fried Phenotype 评分 2 年内增加 ≥1 分 (恶化)
  - 人群: 60+, 社区居住, 基线非衰弱 (Fried <3)
  - 排除: 严重认知障碍 (MMSE<18), 基线 ADL 严重依赖

数据可用性 (from data-engineer):
  - CHARLS Waves 1-4
  - 核心文件: Health_Status_and_Functioning + Biomarker + Blood
  - 2011 无握力数据! 需从 2013 开始或以其他指标替代
  - 血液数据仅部分受访者 (~60% have blood)

请设计完整的建模方案。
```

### Reasoning (Chain of Thought)

```
1. PICO-ML 映射:
   - 临床问题: "预测 2 年内衰弱恶化"
   - ML 任务: 这是一个二分类问题 (恶化 vs 稳定/改善)
   - 也可以考虑有序多分类 (改善/稳定/恶化)，但二分类更直接
   → 主任务: 二分类

2. 数据约束分析:
   - 2011 无握力 → 不能同时用 2011 做基线 + 用握力做特征
   - 方案 A: 用 2013 做基线 (有握力)，预测 2015 → 但样本量减少
   - 方案 B: 用 2011 做基线，弃用握力 → 损失关键预测因子
   → 推荐 方案 A (2013→2015), 2011→2013 做敏感性

3. 样本量评估:
   - CHARLS 2013: ~15,000 参与者
   - 60+ 且基线非衰弱: 预估 ~6,000-7,000
   - 2年完成随访: ~5,000-6,000
   → 样本量足够支撑 XGBoost

4. 模型选择:
   - 表格数据 + 预计 ~80 特征 + N≈5,000
   → 基线: Logistic Regression + LASSO
   → 主力: XGBoost (对表格数据表现稳定, 非线性交互)
   → 不推荐深度学习 (N 不够大, 容易过拟合)

5. 验证策略:
   - 内部: 10-fold nested CV
   - 时间验证: 2015→2018 (不同 wave 对)
   → 最佳实践: 在 1 个时间窗训练, 在后续时间窗验证
```

### Output

```markdown
## 建模方案 — CHARLS 衰弱转换预测

### 1. 临床→ML 映射
| | |
|---|---|
| 临床问题 | 预测社区老年人 2 年内衰弱恶化 |
| ML 任务 | 二分类 (恶化=1, 稳定/改善=0) |
| 结局定义 | Fried Phenotype 总分增加 ≥1 |

### 2. 数据策略
**主分析**: 2013 (基线) → 2015 (结局)
**时间验证**: 2015 (基线) → 2018 (结局)

**理由**: 2013 是首次同时包含握力和步速的 wave，是衰弱研究的最佳基线。

### 3. 预期样本量
- 2013 60+ 岁且基线非衰弱: ~6,500
- 2年完成随访且有结局数据: ~5,500
- 含血液数据子集: ~3,300 (用于含实验室特征的敏感性分析)

### 4. 候选特征 (78个)
| 类别 | 数量 | 示例 |
|------|------|------|
| 人口学 | 6 | 年龄、性别、教育、婚姻、城乡、独居 |
| 临床 | 12 | 慢病数量、多重用药、自评健康、疼痛... |
| 功能 | 8 | 握力、步速、ADL、IADL、平衡、坐站... |
| 认知 | 5 | MMSE总分、回忆、画图、定向... |
| 心理 | 5 | CES-D-10、生活满意度、孤独感... |
| 社会 | 5 | 社会参与、社会支持、婚姻满意度... |
| 生活方式 | 5 | 吸烟、饮酒、体力活动、BMI、睡眠... |
| 实验室 | 10 | Hb、CRP、HbA1c、Alb、eGFR、血脂... |
| 其他 | 22 | 听力、视力、跌倒史、住院史、门诊次数... |

### 5. 模型方案
| 层级 | 模型 | 用途 |
|------|------|------|
| Baseline 1 | LogReg (年龄+性别) | 性能锚点 |
| Baseline 2 | LASSO LogReg (全特征) | 线性方法上限 |
| **主模型** | **XGBoost (Bayesian 调优)** | 主力模型 |
| 探索 | LightGBM | 与 XGBoost 对比 |

### 6. 评估方案
| 维度 | 方法 |
|------|------|

| 可解释性 | SHAP (全局 + 交互 + 个体 waterfall) |

2. 不同时间窗: 2011→2013 (无握力版)
3. 不同结局定义: FI 替代 Fried
4. 年龄分层: 60-69 / 70-79 / 80+
5. 加入死亡作为竞争事件 (Fine-Gray)

### 8. 关键风险
| 风险 | 严重度 | 缓解措施 |
|------|--------|----------|
| 2011 无握力限制敏感性分析 | 中 | 明确标注，补充无握力版模型 |
| 失访导致选择偏倚 | 中 | 比较失访vs完成随访者的基线特征 |
| 血液数据仅60%可用 | 中 | 主模型不含实验室特征，含实验室的做子集分析 |
```

## Example 2: 快速回答方法选择问题

### Input

```
任务类型: 用 CHARLS 数据做多病共存的亚型发现
数据: ~8,000人, 15种慢性病的 0/1 矩阵 + 年龄性别等连续变量
问题: 用 K-means 还是 Latent Class Analysis？
```

### Reasoning

```
1. 分析输入:
   - 变量: 二分类 (15种疾病的0/1) + 连续变量 (年龄等)
   - 这是"混合变量"聚类场景
   
2. 方法比较:
   - K-means: 仅适用于连续变量，不适用于0/1矩阵
   - K-prototypes: 处理混合变量 (二分类+连续) ✓
   - LCA (Latent Class Analysis): 假设存在潜在离散类别，概率模型，适合0/1为主的数据 ✓✓
   - 自编码器+聚类: 高维场景更合适，此处非必需
   
3. 推荐:
   - 主推荐: LCA (因为主要特征是15个0/1疾病变量)
   - 备选: K-prototypes (如果想同时纳入连续变量)
   - 确定簇数: BIC + 临床可解释性 (最重要!)
```

### Output

```
推荐 Latent Class Analysis (LCA)，理由:

1. 你的主要特征 (15种疾病) 是二分类变量，LCA 天然适合对离散变量的潜在类别建模
2. LCA 给出每个参与者属于每个类别的概率，比硬聚类更适合临床解释
3. 如果还需要纳入年龄等连续变量，可以:
   - 方案 A: LCA 仅用疾病变量聚类, 然后描述各亚型的年龄分布
   - 方案 B: 使用 K-prototypes 同时聚类

确定最佳簇数:
  - 统计指标: BIC (LCA) / Silhouette (K-prototypes)
  - 临床标准: 每个亚型应≥10%样本量, 亚型间应有明确临床区分度
  - 最终决策: 由 clinical-researcher 审阅各亚型是否"说得通"

R 包: poLCA (基础) / tidyLPA (tidyverse 友好)
```

## Example 3: DL 方法选型 — 衰弱轨迹时序预测

### Input

```
任务: CHARLS 5波次面板数据, 预测老年人衰弱轨迹

数据:
  - 2011/2013/2015/2018/2020 五个波次
  - 每波次 ~30个特征 (人口学+功能+心理+慢病+生活方式)
  - 2011/2013/2015 有 Biomarker (握力/步速等)
  - 2018/2020 仅问卷数据
  - ~26000 人用于训练

研究问题: 能否用前3波次预测后2波次的衰弱轨迹?
硬件: MacBook Pro M4 (统一内存 16-24GB, MPS 后端)
```

### Reasoning

```
1. PICO-ML 映射:
   - 临床问题: "预测5波次的衰弱轨迹"
   - 这是一个多步时序预测问题 (sequence-to-sequence)
   - 输出: 未来2个时间点的4维目标 (Fried评分+组分)

2. 是否需要 DL?
   - N=26000 > 10000 → DL 可行 ✓
   - 数据结构 = 时序 (天然有序列结构) → DL 有优势 ✓
   - 特征维度 F=30, 序列长度 L=3-5 → 中等规模 ✓
   - 经典 baseline: GEE / Mixed Effects Model
   → 推荐: 经典 baseline + DL 对比

3. 架构选择:
   - 序列长度 3-5 (短) → LSTM/GRU 足够, 不需要 Informer
   - 双向 LSTM + Attention 池化 → 适合中等序列
   - Informer 适用场景是 L>96 的长序列, 这里不需要
   → 主推荐: 2层 BiLSTM + Attention

4. M4 可行性:
   - LSTM 2层 hidden_dim=256 → 参数量 ~3.5M ✓
   - batch_size=32 在 M4 上没问题 ✓
   - 训练时间估算: ~30分钟

5. 不推荐 Informer 的理由:
   - CHARLS 序列长度仅 3-5 波次
   - Informer 的 ProbSparse Attention 优势在长序列 (L>96)
   - 短序列上 LSTM 性能相当, 且训练更快更稳定
```

### Output

```markdown
## 建模方案 — CHARLS 衰弱轨迹时序预测

### 1. 临床→ML 映射
| | |
|---|---|
| 临床问题 | 预测老年人多波次衰弱轨迹 |
| ML 任务 | 多步时序预测 (sequence-to-sequence) |
| 输入 | 前3波次 (2011/2013/2015), 30维特征 |
| 输出 | 后2波次 (2018/2020), 4维目标 (f_score_3item, wl, exh, inactive) |
| 预测窗口 | 3-7 年前瞻预测 |

### 2. 方法设计
| 层级 | 模型 | 用途 |
|------|------|------|
| Baseline 1 | 线性混合效应模型 (GEE) | 经典纵向方法锚点 |
| Baseline 2 | XGBoost (单步预测) | 经典 ML 锚点 |
| **主模型** | **2层 BiLSTM + Attention 池化** | 捕获时序依赖 |
| 探索 (可选) | GRU (轻量对比) | 验证 LSTM 是否 overkill |

### 3. 为什么不推荐 Informer/Transformer
- CHARLS 序列长度 3-5 波次 (短序列)
- Informer 的设计优势在 L>96 的长序列预测
- 短序列上 LSTM 性能相当, 且参数量更小、训练更稳定
- 如果未来扩展到逐月或逐周数据, 再考虑升级到 Informer

### 4. 硬件约束 (M4 Pro)
- BiLSTM ~3.5M 参数在 M4 上完全可训练
- 使用 MPS 混合精度加速, 预估训练时间 ~30分钟
- batch_size=32 适配统一内存

### 5. 评估方案
| 维度 | 指标 |
|------|------|
| 预测精度 | MSE, MAE (分波次报告) |
| 方向准确率 | 衰弱恶化 vs 改善的方向预测准确率 |
| 校准 | 预测 Fried 评分 vs 实际的 Spearman ρ |
| 基线对比 | 与 GEE 和 XGBoost 的 ΔMAE |

### 6. 敏感性分析
1. 仅用问卷特征 (允许扩展到2018/2020无Biomarker的场景)
2. 分性别和年龄组
3. 不同输入-输出长度组合 (2→2, 3→1, 4→2)
```
