# Few-Shot 示例 — Clinical Researcher Agent

## Example 1: 临床问题操作化

### Input (from Orchestrator)

```
项目: CHARLS 衰弱转换预测

需求: 将"预测社区老年人在 2 年内衰弱恶化"这个临床问题
操作化为可计算的研究方案。需要明确:
1. 目标人群的纳入排除标准
2. CHARLS 中 Fried Phenotype 5项的具体变量映射
3. 预测窗口的设计考虑
4. 模型性能的临床效用阈值
```

### Reasoning (Chain of Thought)

```
1. 目标人群:
   - CHARLS 是 45+ 社区人群，但衰弱研究通常关注 60+
   - 需排除基线已衰弱的 (因为他们已经"恶化到顶"了)
   - 严重认知障碍者代理受访的认知数据不可靠
   → 纳入: 60+, 基线 Fried<3
   → 排除: MMSE<18, 基线 ADL≥3 项依赖

2. Fried Phenotype 的 CHARLS 映射:
   - 翻阅 CHARLS codebook 确定每个变量的可用性
   - 关键问题: 2011 无握力！需标注
   - 体力活动变量在各 wave 的措辞可能不同, 需对齐
   
3. 预测窗口:
   - 2 年是老年医学常用的中期预测窗口
   - 太短 (6月): 事件数少
   - 太长 (5年): 失访多, 且与基线年龄高度共线
   
4. 临床效用:
   - 筛查场景: 灵敏度≥0.80, 不想漏掉高风险老人
   - NNS<10: 筛查不到10人就能发现1例真阳性
   - 假阳性可以接受 (后续可用 CGA 确认)
```

### Output

```markdown
## 临床问题操作化 — CHARLS 衰弱转换预测

### 1. 目标人群
| | |
|---|---|
| 纳入标准 | ① CHARLS 参与者 ② 基线年龄 ≥60 ③ 基线 Fried <3 (非衰弱) ④ 2年后完成衰弱评估 |
| 排除标准 | ① 基线严重认知障碍 (MMSE<18, 代理受访) ② 基线 ADL ≥3 项需要帮助 ③ 2年内死亡 (主分析排除, 敏感性分析作为竞争事件) |

### 2. Fried Phenotype — CHARLS 变量映射

#### 2a. 体重下降
| | |
|---|---|
| CHARLS 变量 | da049: "过去一年体重变化" |
| 编码 | 体重下降 ≥5kg 或 ≥5% = 1 |
| 适用 Wave | 2011, 2013, 2015, 2018 |
| 注意事项 | 自报而非客观测量; 区分"刻意"和"非刻意"减重较困难 |

#### 2b. 疲乏
| | |
|---|---|
| CHARLS 变量 | CES-D: dc011 ("我做事费力") + dc012 ("我提不起劲") |
| 编码 | 任一问题回答"大部分时间(3-4天)"或"全部时间(5-7天)" = 1 |
| 适用 Wave | 全 wave |
| 注意事项 | CES-D 在 CHARLS 中是 10 项版, 仅这两个问题用于疲乏 |

#### 2c. 握力下降 ⚠️
| | |
|---|---|
| CHARLS 变量 | Biomarker 文件: 握力 (kg), 左右手各测 2 次 |
| 编码 | 取优势手最大值, 按性别+BMI 四分位, 最低 20% = 1 |
| 适用 Wave | **2013, 2015** (2011 无握力!) |
| 注意事项 | 2011 需特殊处理: 排除握力或使用替代指标 |

#### 2d. 步速减慢
| | |
|---|---|
| CHARLS 变量 | Health_Status: 2.5m 计时行走 (秒) |
| 编码 | 转换为 m/s, 按性别+身高四分位, 最低 20% = 1 |
| 适用 Wave | 2013, 2015, 2018 (具体需确认) |
| 注意事项 | 无法行走者自动记为 1 分; 需注意用辅助器具的影响 |

#### 2e. 体力活动低
| | |
|---|---|
| CHARLS 变量 | da051: 体力活动频率 (从不/偶尔/经常) + 强度 (轻度/中度/重度) |
| 编码 | 频率×强度 = 每周代谢当量, 按性别四分位, 最低 20% = 1 |
| 适用 Wave | 全 wave ⚠️ 各 wave 的类别措辞有差异, 需对齐 |

### 3. 预测窗口
- **主窗口**: 2 年 (基线 → 2年后随访)
- **场景**: 社区筛查, 识别高风险老人进行早期干预
- **选择理由**: 2 年足够进行预防性干预 (运动处方、营养支持、药物调整)

### 4. 临床效用阈值
| 指标 | 最低可接受值 | 理想值 |
|------|-------------|--------|
| 灵敏度 | ≥0.80 | ≥0.85 |
| 特异度 | ≥0.60 | ≥0.75 |
| PPV | ≥0.40 | ≥0.50 |
| NPV | ≥0.90 | ≥0.95 |
| NNS | ≤10 | ≤5 |

**解释**: 这是一个筛查模型 (不是诊断模型), 所以灵敏度优先于特异度。
假阳性可以通过后续老年综合评估 (CGA) 确认, 成本可接受。
假阴性的代价更高 (错过预防窗口)。

### 5. 特殊注意事项
1. **2011 wave 握力缺失**: 主分析用 2013→2015, 2011→2013 仅做敏感性
2. **代理受访者**: 认知障碍者的功能评估可能不准确, DATA 中标记后排除
3. **死亡率**: 2年内死亡的老人是"最衰弱"的, 排除他们可能高估模型性能
   → 敏感性分析将死亡作为结局 (竞争风险)
4. **各 wave 体力活动问题变化**: 需仔细对齐, 或使用保守的二分法编码
```

## Example 2: 模型结果临床审查

### Input

```
任务: 审查 XGBoost 衰弱预测模型的结果

模型输出:
  AUC-ROC: 0.84
  Top 5 SHAP: 步速、年龄、慢性病数、握力、白蛋白
  其中"教育年限"SHAP排名第8 (方向: 高教育 → 低衰弱风险)
  年龄加速 (AgeAccel) 排名第6
  
校准曲线: 高风险区间 (predicted probability>0.6) 系统低估了实际风险
```

### Reasoning

```
逐个审查六条标准:

1. 预测因子合理性:
   - 步速、握力 → 合理, 是衰弱的核心标志
   - 慢性病数 → 合理
   - 白蛋白 → 合理, 营养/炎症标志物
   - 教育年限 → 合理, 社会经济+健康素养+健康行为的中介
   - 年龄加速 → 需要确认定义
   
2. 效应方向:
   - 步速/握力/白蛋白 高 → 风险低 ✓
   - 年龄/慢性病数 高 → 风险高 ✓
   
3. AUC 0.84: 满足筛查需求 (>0.80)

4. ⚠️ 校准度问题:
   - 高风险区间系统低估 → 这意味着模型对于它说"高风险"的人,
     实际风险比预测的更高
   → 对于筛查来说可能是好事 (模型保守但实际更危险的人被识别出来)
   → 但需要报告和讨论

5. 决策曲线需要看到才能评估

6. 年龄混淆: 
   - 年龄排名第2, 这是合理的 (年龄是衰弱最强的预测因子之一)
   - 需要看在年龄匹配亚组中模型是否仍有区分度
```

### Output

```markdown

### 预测因子合理性
- ✓ **合理 (top features)**: 步速、年龄、慢性病数、握力、白蛋白 — 均与衰弱病理生理机制一致
- ✓ **教育年限 (rank #8)**: 合理，社会经济地位通过多路径影响健康
- ? **年龄加速 (rank #6)**: 需确认计算方式。如果是残差(表观年龄-实际年龄), 
  需注意: 年龄加速本身可能包含衰弱信号, 作为预测因子存在循环论证风险

### 性能评估
- **区分度**: AUC=0.84, 满足社区筛查需求 ✓
- **校准度**: ⚠️ 高风险区间 (pred>0.6) 系统低估风险
  - 含义: 模型对高风险者的风险估计偏保守
  - 影响: 作为筛查工具影响不大 (这些人在高风险组, 会被筛查出来)
  - 但若用于个体化风险告知, 可能低估某些老人的真实风险
- **临床净获益**: 待给出决策曲线后评估

### 循环论证风险提示
年龄加速 (AgeAccel) 作为预测因子需要谨慎:
- 如果 AgeAccel 的计算已经包含了衰弱相关的甲基化信号, 
  用它来预测衰弱就存在循环论证
- 建议: ① 明确 AgeAccel 的定义 ② 敏感性分析: 不含 AgeAccel 的模型性能

### 总体判断
- **推荐**: 继续推进, 但需修复校准问题 + 澄清 AgeAccel
- **下一步**: 请 ml-engineer 给出校准曲线 + 决策曲线; 
  在讨论部分应坦诚讨论高风险区间的校准不足
```
