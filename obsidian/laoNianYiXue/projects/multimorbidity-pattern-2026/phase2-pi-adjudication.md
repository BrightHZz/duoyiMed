## PI 裁决

### 决策 1: 多重用药是否纳入核心预测特征?
**裁决**: ✅ **A. 纳入** — 前提: data-engineer 确认 da008_* 变量覆盖率 >50%

**理由**:
- 多重用药是老年医学核心评估维度 (Beers Criteria/STOPP 标准的基础)
- clinical-researcher 的论据有力: 多重用药是共病负担的直接下游, 比疾病列表更能反映"疾病严重度"
- 参考 prostate-cancer-2026 经验: CCI + 多重用药的交互项有预测价值
- 如果 da008_* 覆盖率 <50% → limitation 中声明, 不作为核心特征

---

### 决策 2: 主要结局报告策略?
**裁决**: ✅ **A. 三个结局分开报告为主** — 复合结局作为敏感性分析

**理由**:
- clinical-researcher 的论据有说服力: 死亡/ADL失能/住院的临床路径和干预靶点不同, 合并会模糊信号
- 住院确实受医疗可及性影响 → Discussion 中必须讨论此局限性
- 如果死亡事件数不足 (events < 200) → 采用 LASSO 降维至 15 个特征, 不改用复合结局
- 复合结局 (死亡 OR ADL OR 住院) 作为预注册的敏感性分析报告

---

### 决策 3: LCA 聚类数上限?
**裁决**: ✅ **A. 统计搜索 k=2:8, PI 最终选择时施加临床约束**

**理由**:
- 统计上不应预先限制数据 → 可能最优解在 k>6 (虽然罕见)
- PI 在模型选择阶段施加临床约束:
  - 最终选择的 k ≤6
  - 最小类占比 ≥5%
  - 每个类必须有临床可区分的特征 (非"噪声类")
- 如果 BIC 持续下降到 k=8 且 k=6/7/8 的边际改善 <2% → 选 k≤6 中 BIC 最低的

---

### 决策 4: 自报慢病偏倚的敏感性分析?
**裁决**: ✅ **A. 是** — 如果 CHARLS Wave 1 有实测血压/血糖

**理由**:
- 中国未诊断高血压约 30%、未诊断糖尿病约 50% → 自报数据系统性偏低
- 如果 Biomarker 文件有实测 SBP/DBP 和 HbA1c → 可构建"实测高血压"和"实测糖尿病"变量
- 敏感性分析: 用实测定义替代自报定义, 重新 LCA → 比较聚类一致性
- 如果实测数据不可用 → limitation 中诚实声明, 引用中国未诊断率文献

## 裁决后合并方案

基于 PI 裁决，最终 Phase 2 方案如下:

### 确定事项

| 事项 | 最终决定 |
|------|---------|
| Stage 1 方法 | LCA (主) + K-prototypes (稳健性) |
| Stage 2 方法 | XGBoost (主) + Logistic (基线) |
| LCA 聚类搜索 | k=2:8, PI 临床约束选 ≤6 且最小类 ≥5% |
| 核心预测特征 | 年龄, 性别, LCA标签, CCI, **多重用药**, ADL |
| 缺失处理 | MICE (m=10) 主分析, CCA 敏感性 |
| 结局报告 | **三个结局分开报告为主**, 复合结局敏感性 |
| 增量价值指标 | ΔAUC + NRI (category-free) + IDI |
| 验证 | 10-fold nested CV + Wave 2→4 时间验证 |
| 敏感性分析 | 5+ 项 (含实测慢病定义的敏感性) |
| 亚组 | 年龄/性别/城乡/CCI 分层, 交互项检验 |
