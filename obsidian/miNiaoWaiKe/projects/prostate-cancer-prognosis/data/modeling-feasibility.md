## 2. 方法可行性

### 方案 A: 30天全因死亡预测 (推荐)

**技术难度**: ★★★☆☆ (中等)

- **算法选择**:
  - Baseline: Logistic Regression (线性基准)
  - 主力模型: XGBoost (处理混合类型+缺失值 + 类别不平衡)
  - 对比模型: Random Forest, Gradient Boosting (LightGBM)
- **特征工程**:
  - 实验室时序特征: 入院前 24h/72h 的均值、趋势、最后一次值
  - 合并症评分: Charlson Comorbidity Index (从 ICD 编码计算)
  - 用药特征: 关键药物类别计数 (抗凝、抗生素、阿片)
- **验证**: 5-fold CV (按患者分层，避免同一患者多次入院泄漏)
- **性能预期**: AUC 0.75–0.85 (基于类似 EHR 短期死亡预测文献)

- 预测因子: ~15–20 features
- 最小正例数: ~100–150 events
- MIMIC-IV 前列腺癌队列:
  - 住院死亡 event rate: TODO (需提取)
  - 若 30天死亡率 ~5%: ~100–140 events / 2,000–2,800 admissions → 处于边界
  - 若为复合结局 (死亡+ICU): 预期更多 events → 更可行

### 建议

- 使用复合结局 (30天死亡 OR ICU 入院) 增大正例数
- 或使用 90天窗口
- 特征数控制在 15-20 (满足 events per variable ≥ 10)
